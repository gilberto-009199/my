O CUPS, que faz parte da Base Padrão do Linux, está ativado. Para parar o serviço: 

sudo /etc/init.d/cupsys stop

Para desativá-lo da inicialização: 

update-rc.d -f cupsys remove
update-rc.d cupsys stop 20 2 3 4 5 .

Outra maneira de parar este servidor de impressão é abrir um shell de terminal ctrl + alt + t e digitar sudo service cups stop

Isso interromperá o servidor e fechará a porta, mas será reiniciado em uma reinicialização. Você também pode reiniciar ou iniciar o serviço dessa forma simplesmente substituindo restart ou start , em que stop está no comando. : D

Testing for this can be done with netstat -an --inet | grep LISTEN | grep -v 127.0.0.1: on a fresh install.

(Isto cobre o ipv4. Um comando diferente é necessário para o ipv6).

Eu acredito que você vai encontrar uma nova instalação do Ubuntu está escutando no endereço 127.0.0.1, que só é acessível a partir do seu próprio computador. Se você digitalizar, por exemplo nmap localhost , isso localizará o listener IPP do host local. Mas, se você verificar o endereço IP da sua interface de rede, não encontrará nenhum ouvinte.
